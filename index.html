Overview

High-level, dynamic, untyped, interpreted programming language suitable for object-oriented and functional programming styles

The Good parts

  • Lambdas
  • Dynamic objects
  • Loose typing
  • Prototypal inheritance
  • Single threaded execution model
  • Object literal (JSON)

The Bad parts

  • Global variables
  • The most misunderstood language
  • Semicolon insertions
  • No block scope
  • No standard library
  • with/eval/typeof/null vs undefined

Client-side vs server-side dichotomy

Closer overview

No compilator - only interpreter

JS is interpreted. But some browsers do optimize code - i.e. chrome compiles js to native code

JSLint

Garbage collector

Single threaded execution model

  • There is only single thread for javascript in browser
  • No waiting, no locking...
  • no blocking

No standard library - only pure javascript and environment

Language is case-sensitive

Many reserved words, only couple of them are in use (+ global variables)

Optional semicolon

Can be omited between lines and js will insert it itself

Identifiers and keys

  • [a-zA-Z_$][a-zA-Z_$0-9]
  • no dash etc
  • the same goes for object properties names
  • if using square bracket notation - everything is permitted

Types

Primitive types

  • number
  • string
  • boolean
  • null
  • undefined (kind of)

Object types

everything that is not a primitive type is an object

Boolean

  • no surprise - immutable, ordered sequence of characters
  • in ES3 string literals must be oneliner, ES5 permits line breaks

Number

  • all numbers are 64-bit floating point IEEE-754 (java double)
  • no integer/double distinction
  • arithmetics don't raise error in case of overlow/underflow/division by zero
  • edge cases - Infinity/NaN
  • NaN != NaN (the only case where x!=x)

String

  • true/false
  • falsy values: undefined, null, 0, -0, NaN, ''
  • truthy values: everything else (even empty array)

Date

Let's do it java style - zero based months, 1 based days.

null

It's not an object, but `typeof null` returns always 'object'

undefined

  • absence of property
  • unitialized value
  • it's global value, it can be set to whatever (in ES3)

wrapper objects for primitives

equality

  • === strict equation (identity operator)
  • ==

Slide Backgrounds

Set data-background="#007777" on a slide to change the full page background to the given color. All CSS color formats are supported.

Image Backgrounds

<section data-background="image.png">

Repeated Image Backgrounds

<section data-background="image.png" data-background-repeat="repeat"
      data-background-size="100px">

Background Transitions

Pass reveal.js the backgroundTransition: 'slide' config argument to make backgrounds slide rather than fade.

Background Transition Override

You can override background transitions per slide by using data-background-transition="slide".
